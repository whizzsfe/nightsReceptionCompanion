# Night Receptionist Companion Suite

A browser-based toolkit for hotel front-office night staff. Runs entirely from a local file — no server, no internet connection required during a shift.

---

## What's included

| Tool | What it does |
|---|---|
| **Night Dashboard** | Live clock, shift stats, lone-worker check-in, incoming handover banner |
| **Shift Checklist** | Pre-Audit and Post-Audit tabs with progress tracking, day-of-week filtering, and a printable handover sheet |
| **Notes & Handover** | Shift notes, outgoing handover, and incoming handover from the previous shift |
| **Night Log** | Log issues and floor walks during the shift; export to JSON; quick-log from anywhere via the ⚡ button |
| **Shift History** | All completed shifts with aggregate stats, incident charts, and a filterable all-incidents table |
| **AI Coach — Aria** | Keyword-triggered scripts for common hospitality scenarios (no internet required) |
| **Group Check-in** | Print-ready group arrival list with bulk name import |
| **Sign Generator** | A4/A3 ISO-style signs with custom background, logo, and text |
| **Daily Info Board** | 7-day rolling stats board, VIP list, staff assignments, and Principle of the Day |
| **Motivation** | Quote carousel and excellence tips for lone workers |

## Setup — getting started

### What you need
- A Windows PC with access to the OneDrive sync folder
- Island Enterprise Browser (or any modern Chromium-based browser)
- No installation, no internet, no admin rights required

### Files

| File | Purpose |
|---|---|
| `hotelcompanion_suite.html` | **The main app** — open this to start a shift |
| `hotelcompanion_editor.html` | **Content editor** — managers use this to customise checklist items and AI scripts |
| `hotelcompanion_items.js` | Your property's checklist content *(generated by the editor — see below)* |
| `hotelcompanion_scripts.js` | Your property's AI Coach scripts *(generated by the editor — see below)* |
| `hotelLogo.jpg` | Hotel logo displayed in the app and on printed documents |

All files must be kept **in the same folder**. OneDrive keeps them in sync across all front-desk PCs automatically.

### First-time setup

1. Copy all files into your shared OneDrive folder.
2. Open `hotelcompanion_suite.html` in the browser.
3. Fill in the **Setup Profile** form (your name, role, and shift type).
4. Optionally open **🎨 Customise** in the sidebar to set your hotel name, logo, and accent colour.

## Customising checklist items and AI scripts

These are changed using the **Content Editor** (`hotelcompanion_editor.html`) — no coding required.

1. Open `hotelcompanion_editor.html` in the browser.
2. Edit, add, reorder, or delete items across the **Pre-Audit**, **Post-Audit**, and **AI Scripts** tabs.
3. Click **Export Items (.js)** — save the file as `hotelcompanion_items.js` in the same folder as the main app.
4. Click **Export Scripts (.js)** — save as `hotelcompanion_scripts.js` in the same folder.
5. Reload `hotelcompanion_suite.html` — your changes are live.

> **To import your night operations manual into AI Scripts:** Open the **AI Scripts** tab in the editor, click **Import from PDF**, select your PDF, review the extracted draft scripts, and export.

> **To load back a file you previously exported:** Click **Import / Load Existing** in the editor and select the `.js` file.

## During a shift

- Use the **Shift Checklist** to work through Pre-Audit and Post-Audit tasks.
- Tap **⚡** (bottom-right corner) to quickly log an incident from anywhere.
- Use **Night Log** for detailed issue recording and floor walks.
- Click **AI Coach — Aria** and type a scenario (or tap a suggestion chip) for instant scripts.
- Toggle **Persist Data** in the sidebar to save session data between page reloads. On shared terminals, leave this off.

---

## Ending a shift

1. Fill in the **Handover** notes in the Notes & Handover section.
2. Click **End Shift** in the sidebar.
3. Confirm — your shift is saved to history, the handover is queued for the next user, and session data is cleared.

## Data & privacy

- All data is stored locally in the browser's `localStorage` — nothing is sent to any server.
- Shift history and profile data persist across sessions.
- Session data (notes, checklist ticks, log entries) is only saved if the **Persist Data** toggle is on.
- Clearing browser data or using a different browser will lose any saved data.

## Printing

- **Handover sheet:** Notes & Handover → Print Handover
- **Checklist:** Shift Checklist → Print
- **Group check-in list:** Group Check-in → Print
- **Signs:** Sign Generator → Print (A4 or A3)
- **Daily Info Board:** Daily Info Board → Print (A3 landscape recommended)

In the browser print dialog, set **Margins to None** and tick **Print backgrounds** for the best results.

## Data Persistence — when does data save?

The **Persist Data** toggle in the sidebar controls how session data behaves. It defaults to **Off**.

| What data | When it saves |
|---|---|
| Staff profile (name, role, shift type) | **Always** — regardless of toggle |
| Shift history | **Always** |
| Pending handover for the next shift | **Always** |
| Brand customisation (hotel name, logo, colour) | **Always** |
| Shift start time | **Always** |
| Checklist ticks | Only when toggle is **On** |
| Shift notes & handover notes | Only when toggle is **On** |
| Night Log entries | Only when toggle is **On** |
| Night Audit step state | Only when toggle is **On** |
| Daily Info Board data | Only when toggle is **On** |

**Recommended setting for shared terminals:** Leave the toggle **Off**. Each person who opens the app starts with session data cleared — the right behaviour for a front desk PC used by multiple staff across a 24-hour rota.

**Turn the toggle On** if you are the only person using the PC during your shift and want your progress saved against accidental page reloads.

Turning the toggle **On** immediately saves the current session data. Turning it back **Off** does not delete what was already saved — it just stops new changes being written.

## OneDrive folder structure

All files should live in a single shared OneDrive folder (e.g. `NightShifts`). OneDrive syncs the folder to every front-desk PC automatically.

```
NightShifts/
├── hotelcompanion_suite.html        ← Main app (open this to start a shift)
├── hotelcompanion_editor.html       ← Content editor (managers only)
├── hotelcompanion_items.js          ← Your custom checklist content (generated by editor)
├── hotelcompanion_scripts.js        ← Your custom AI scripts (generated by editor)
├── hotelLogo.jpg                    ← Hotel logo
└── shiftLogs/                       ← Auto-saved shift JSON files (one per shift)
    ├── shift_2026-04-08.json
    └── shift_2026-04-09.json
```

The `shiftLogs/` sub-folder is created automatically when you configure auto-save (Set folder → select or create the folder). The Monthly Report section reads from this folder to generate charts.

> `hotelcompanion_items.js` and `hotelcompanion_scripts.js` are property-specific and are **not** committed to version control. Use the `.example.js` files as starting points.

## Cross-device behaviour

All data stored by the app lives in the **browser's localStorage on each individual PC**. There is no shared cloud database.

This means:
- **Shift history** built up on PC 1 is not visible on PC 2.
- **Brand customisation** (hotel name, logo, colour) set on PC 1 must be set again on PC 2.
- **Persist Data toggle** state is per-device.

What *is* shared across PCs (via OneDrive file sync):
- The app itself, the editor, the logo, and the content files (`.js`).
- The handover from one shift to the next — this is written at End Shift and loaded by whoever opens the app next, on any PC.

---

## Shift history storage

History is stored in `localStorage` on the current device. The app keeps up to **365 entries** (approximately one year). When the limit is reached, the oldest entries are trimmed automatically.

Photo attachments in Night Log entries are **stripped** before saving to history to keep storage usage low. The photos are still visible during the active shift but are not included in the history record.

Clearing browser data (e.g. `chrome://settings/clearBrowserData`) will erase all localStorage including shift history, profile, and brand settings.

## Night Log — field reference

| Field | Required | Notes |
|---|---|---|
| Floor | Yes | Ground / First / Second / Third / Roof |
| Category | Yes | Noise Complaint, Maintenance, Security, Guest Services, Medical, Other |
| Severity | Yes | Low / Medium / High |
| Title | Yes | Short summary — shown in the entry list |
| Description | No | Full detail of what happened and actions taken |
| Reporter | No | Name of the person who reported the issue |
| Photo | No | Single image attached via file picker; resized to max 800 px before saving |

**Floor walks:** Press the floor button after each walkaround on that floor. Best practice is a minimum of **3 walks per floor** per night. The button shows the running count.

**All entries are cleared when you click End Shift.** Export the JSON file first if you need the data for the Monthly Report.

## Troubleshooting

**My ticks / notes are gone after I reloaded the page.**
The Persist Data toggle was off. Session data is only saved between reloads when the toggle is on.

**AI Coach shows "Smart Defaults Active" but I exported scripts.**
Ensure `hotelcompanion_scripts.js` is in the **same folder** as `hotelcompanion_suite.html` and reload the page.

**My hotel name / logo / accent colour are gone.**
Browser data was cleared, or you are on a different device or browser. Settings are per-device; re-apply via 🎨 Customise.

**The logo doesn't appear.**
The file must be named `hotelLogo.jpg` and placed in the same folder as the app. File names are case-sensitive on some systems.

**Printing doesn't look right.**
In the browser print dialog: set Margins to **None**, tick **Print backgrounds**, and match the paper size to the setting in the app.

**"No text extracted" when importing a PDF in the editor.**
The PDF is a scanned image, not a text-based document. A digitally created (Word-exported) PDF is required for text extraction.

**The app isn't picking up my exported items.js file.**
Check the file is named exactly `hotelcompanion_items.js` (browsers sometimes append ` (1)` on download). Place it in the same folder as the app and reload.

- **Daily board:** Daily Info Board → Print
- **Shift report:** Shift History → Print Report

All print views suppress navigation chrome automatically.

## Troubleshooting

| Problem | Fix |
|---|---|
| Checklist items or AI scripts not updating after export | Make sure the `.js` file is in the **same folder** as `hotelcompanion_suite.html`, then hard-reload (`Ctrl+Shift+R`) |
| Logo not showing | Rename your logo file to `hotelLogo.jpg` and place it in the same folder |
| Shift history missing on another PC | History is per-device only. Use **Export Night Log** during a shift to preserve records, or use the Shift History sync feature |
| AI Coach says "Smart Defaults Active" | `hotelcompanion_scripts.js` is missing or hasn't been exported yet — use the editor to generate it |
