Help & Guide
How to use every part of this app. Click a heading to expand.
🚀 Getting Started

When you open the app for the first time, a Setup Profile box will appear. Enter your name, your role, and your shift type — this is saved permanently to this device and shows in the sidebar.

You can update your profile at any time by clicking your name card at the top of the sidebar.

The shift timer in the sidebar starts automatically and keeps running while the page is open. If you reload the page, the timer picks up from where it left off.

All files (suite, editor, items.js, scripts.js, logo) must be kept in the same folder for customisations and the logo to load correctly.
🌙 Night Dashboard

The dashboard gives you an at-a-glance view of the current shift:

  • Checklist progress — percentage of items ticked across both tabs.
  • Incidents logged — number of issue entries in the Night Log this shift.
  • Shift streak — total completed shifts stored in history on this device.
  • Tonight's tip — rotates daily, keyed to your shift start date.

The Quick Actions cards navigate directly to key sections.

The Lone Worker Safety card has a "I'm OK — Check In" button. Press it regularly during your shift. It records the time so a manager can confirm you were active.

If a handover was left by the previous shift, a gold banner will appear here. Read it, then dismiss it when done.
✅ Shift Checklist

The checklist has two tabs:

  • Pre-Audit — tasks completed on arrival and up to running the night audit (arrival procedures, cash float, system checks).
  • Post-Audit — tasks completed after the audit runs (reports, banking, handover preparation).

Items marked Required are highlighted in yellow. These should be completed before signing off that phase.

Some items show a day badge (e.g. "Mon only", "Last night of month"). Items that don't apply today are greyed out — you don't need to tick them.

The Run Night Audit card at the end of the Pre-Audit tab navigates to the Night Audit Tracker when you're ready.

To customise items (add, remove, reorder, change day restrictions) use the Content Editor (hotelcompanion_editor.html). No coding required.
🔄 Night Audit Tracker

The Night Audit Tracker is a step-by-step guide through the audit procedure — entirely separate from the shift checklist. It covers PMS steps: posting charges, running the audit, printing reports, reconciling payments, and closing the day.

  • Tick each step as you complete it.
  • Each step has a notes field for logging anything unusual.
  • The progress bar updates in real time.
  • Use Reset Audit to clear all ticks if you need to re-run.
Audit steps and checklist items are independent — completing one does not affect the other.
📋 Night Log

The Night Log is the single place for all incident and observation recording. It has two entry types:

Issue entries — fill in the form at the top of the section:

  • Floor — where the incident occurred.
  • Category — type of incident (noise complaint, maintenance, security, etc.).
  • Severity — Low / Medium / High.
  • Title — a short description (required).
  • Description — full detail of what happened and what action was taken.
  • Reporter — name of the person who reported it.
  • Photo — optional image (file picker only — live camera is not available).

Floor walks — use the floor walk grid at the top. Press the button for a floor after each walkaround. The button shows the walk count for that floor this shift. Aim for at least 3 walks per floor per night.

Entries can be Resolved (marked green), Edited, or Deleted at any time.

Use Export JSON to save the shift's log as a file for the Monthly Report.

The ⚡ button in the bottom-right corner opens a quick log form from anywhere in the app — entries go straight into the Night Log.
All Night Log entries are cleared when you click End Shift. Export the JSON first if you want a record for monthly reporting.
📝 Notes & Handover

There are two text areas in this section:

  • Shift Notes — your working scratch pad for the current shift. Use it for anything you want to track mid-shift that doesn't need to be in the Night Log.
  • Handover Notes — what you want to pass on to the next shift. Write what the incoming team needs to know, then click Print Handover before you leave.

Click Print Handover to print a formatted handover sheet — the sidebar and navigation are hidden automatically.

Notes are only saved between page reloads if Persist Data is on. With the toggle off, reloading the page will lose any unsaved notes.
🤖 AI Coach — Aria

Aria is an offline, keyword-driven script assistant. She does not use the internet or any external AI service — all responses come from built-in scripts or your property's custom scripts file.

To use: Type a scenario in the chat box, or tap one of the suggestion chips. Aria responds within about a second.

Built-in topics include: noise complaints, card declines, intoxicated guests, no-show reservations, night audit errors, fire alarms, medical emergencies, VIP arrivals, lost property, and more.

The status badge in the chat header shows:

  • ● Property Scripts Loaded — your custom hotelcompanion_scripts.js file is present and loaded. Property scripts take priority over built-in ones.
  • ● Smart Defaults Active — the custom file is missing or invalid; built-in scripts are in use.

If Aria has no script for your query, she will suggest three practical steps: contact the Duty Manager, check the night manual, or log the incident.

To add property-specific scripts: open the Content Editor → AI Scripts tab → add or import from your PDF manual → export hotelcompanion_scripts.js to the same folder.
📊 Shift History

Every time you click End Shift, a summary is saved to history on this device. Each entry shows:

  • Date, duration, staff name and role
  • Checklist completion percentage
  • Number of incidents and floor walks
  • Handover notes left at shift end

Aggregate stats (total shifts, average checklist %, incidents by type) are shown at the top.

Click 📋 Load Handover on any entry to push that shift's handover notes back to the incoming-handover banner — useful if a handover was accidentally dismissed.

History lives in this browser on this device only. It is not shared between devices or browsers. The app keeps up to 365 entries; older ones are removed automatically. Clearing browser data will erase everything stored here.
"Clear History" removes all stored entries permanently. The app will ask you to confirm before proceeding.
💾 Data Persistence Toggle

The Data Persistence toggle is in the sidebar below the navigation. It defaults to On.

When OFF: Session data (checklist ticks, notes, Night Log entries) is held in memory only. Closing or reloading the tab loses that data. This is the right setting for a shared terminal — the next person who opens the app starts with a clean slate.

When ON (default): Session data is saved to this browser's localStorage and restored if you reload. Use this when you are the only person using this PC during your shift and want protection against accidental reloads.

The following is always saved regardless of this toggle:

  • Staff profile (name, role, shift type)
  • Shift history
  • Pending handover for the oncoming shift
  • Brand customisation (hotel name, logo, accent colour)
  • Shift start time
📁 Shift Auto-Save (OneDrive Folder)

The Shift Auto-save panel in the sidebar lets you configure a folder where the app automatically writes a JSON file when you end a shift.

Click Set folder and grant browser permission to a folder — typically a sub-folder inside your OneDrive sync folder (e.g. NightShifts\shiftLogs).

Each file is named shift_YYYY-MM-DD.json and contains the full shift summary: notes, checklist state, Night Log entries, and stats.

The Monthly Report section can import these files to generate charts and summaries across multiple nights.

The folder permission is granted by the browser using the File System Access API. You may need to re-grant permission after the browser is closed or after a system security policy update. You will be prompted automatically if permission has lapsed.
Auto-save at End Shift is separate from the manual Export JSON button in the Night Log section. Both produce JSON, but auto-save covers the full shift summary while Night Log export covers log entries only.
🖨️ Printing

Several sections support printing. Use your browser's print function (Ctrl+P) or the on-screen Print button. The sidebar, topbar, and all navigation are hidden automatically when printing.

  • Handover Sheet — Notes & Handover → Print Handover
  • Shift Checklist — Shift Checklist section → Print
  • Group Check-in List — Group Check-in → Print (portrait or landscape)
  • Sign Generator — Sign Generator → Print (A4 or A3)
  • Daily Info Board — Daily Info Board → Print (landscape, A3 size recommended)
In the browser print dialog, set Margins to None and tick Print backgrounds for the best results. For the Daily Info Board, set paper size to A3 landscape.
🎨 Brand Customisation

Click 🎨 Customise at the bottom of the sidebar to open brand settings:

  • Hotel / Property Name — appears in the sidebar, print headers, and AI Coach context.
  • Hotel Logo — upload a JPG or PNG; it is stored as base64 data on this device and shown in the sidebar and on printed documents.
  • Accent Colour — choose from 14 preset swatches or click the colour picker for a custom value. This replaces the gold highlight colour throughout the app instantly.

Click Apply & Save to save changes permanently to this device. Click Cancel to discard any changes.

Brand settings are saved in this browser on this device only. They will need to be set once per PC. Clearing browser data removes them.
🛠️ Content Editor (Managers Only)

The Content Editor is a separate file (hotelcompanion_editor.html) in the same folder. Open it in any browser to customise content without touching any code:

  • Pre-Audit Items — arrival and pre-audit checklist tasks
  • Post-Audit Items — post-audit and close-of-shift tasks
  • Night Audit Steps — the step-by-step audit procedure
  • AI Scripts — property-specific response scripts for the AI Coach

Each item has a drag handle for reordering, a label field, a "Required?" toggle, and a day-restriction dropdown (every day, Mon only, last night of month, etc.).

When finished editing:

  • Click Export Items (.js) → save as hotelcompanion_items.js in the same folder → reload the app.
  • Click Export Scripts (.js) → save as hotelcompanion_scripts.js in the same folder → reload the app.

To import your night operations PDF: AI Scripts tab → Import from PDF → select the file → review draft scripts → export.

To load back a file you previously exported: click Import / Load Existing in the editor and select the .js file.

👥 Night Rota & Contacts

The Night Rota & Contacts section has two tabs:

  • Tonight & Contacts — see who is on shift tonight (pulled from the rota planner), manage on-call contacts, and list emergency/property numbers (alarm company, lift engineer, maintenance, etc.).
  • Rota Planner — a rolling 14-night grid where a supervisor assigns night staff to each shift date. Add your team to the staff list, then select names into each shift slot.

The rota is saved to and loaded from your shared OneDrive folder. Set the folder up on the Dashboard first. The file is named nightRota.json and is shared across all PCs that point to the same folder.

Editing permissions: Only staff with the role Night Supervisor can save changes. Other roles see a read-only view.

Use the Print button to produce a clean printed rota — all navigation chrome is hidden automatically.

Keep on-call contacts up to date at the start of each week so the night team always has the right numbers without hunting through emails.
The rota file lives on OneDrive, not in this browser. Clearing browser data does not remove it — but losing access to the OneDrive folder will.
🏨 Group Check-in

The Group Check-in tool generates a print-ready check-in sheet for group arrivals — coach parties, sports teams, event delegates, or any multi-room booking.

  • Enter the Date and Group / Tour Name at the top.
  • Optionally upload your hotel logo for a branded header.
  • Add guest rows one at a time with + Add Guest, or paste a full list of names into the Bulk Import modal (one name per line).
  • Each row has fields for: Name, Room, Email, Car Registration, and Signature.
  • Toggle Portrait / Landscape to match the paper you want to print on.

Click Print — only the check-in table is printed; all app chrome disappears.

Prepare the sheet during a quiet moment before the group arrives so the desk is ready when the coach pulls up.
🪧 Sign Generator

Design and print professional notices without leaving the app — no design software needed.

  • Paper size — A4 or A3.
  • Orientation — Portrait or Landscape.
  • Background — pick a colour from the palette or upload a background image. Use the opacity slider to overlay a colour wash over the image.
  • Logo — optional upload shown in the top-left corner.
  • Body Text — the main message, with a colour picker for the text colour.
  • Sub-body Text — a smaller line below (e.g. a phone number or instruction), also with its own colour picker.

The live preview updates as you type. Click Print when ready — the preview fills the page and all app chrome is hidden.

Common uses: "Lift Out of Order", "Pool Closed — Maintenance", "Reception — Please Ring Bell", "No Entry — Staff Only".
In the browser print dialog, set Margins to None and tick Print backgrounds so coloured backgrounds print correctly.
📋 Daily Info Board

The Daily Info Board mirrors your physical back-office board. Fill it in at the start of the shift and print it for the team.

  • 7-Night Stats Grid — a rolling seven-night table. Set the date and fill in: Rooms Sold, Guests in House, Occupancy %, Room Revenue, Total Revenue, ADR, RevPAR, Arrivals, Departures, and F&B Covers. Column headers auto-populate from the selected date.
  • VIP Guests — up to 10 rows for Name, Code, and Room.
  • Staff Assignments — Pool Responsible, Pool Tester, and First Aider for AM / PM / Night shifts.
  • Principle of the Day — rotates automatically by day of the week (Be Caring, Be A Storyteller, Be Extraordinary, Be Open-Minded…).
  • Special Notes / Info — free text for anything else the team needs to know tonight.

Board data is saved when Persist Data is on, so you can reload without losing your entries. Click Reset to clear all fields and start fresh for a new night.

Set paper to A3 Landscape in the print dialog for the best fit. A4 Landscape also works if A3 is not available.
🌗 Display Theme

The Display Theme switcher sits at the bottom of the sidebar. Three modes are available:

  • 🌙 Dark — the default. Best for night shifts in dimly lit reception areas.
  • ☀ Light — a light cream-and-gold theme. Useful in bright offices or during day handover meetings.
  • 🖥 Auto — follows your operating system's light/dark preference and switches automatically if that setting changes.

Your choice is saved to this device and restored each time you open the app.

Dark is recommended for overnight use — it reduces glare and is easier on the eyes during a long shift.
⚠️ Troubleshooting

My checklist ticks / notes disappeared after I reloaded.
The Persist Data toggle was off. Turn it on before reloading if you want session data to survive a page reload.

AI Coach shows "Smart Defaults Active" but I exported scripts.
Check that hotelcompanion_scripts.js is in the same folder as hotelcompanion_suite.html. Reload the page after placing the file there.

My hotel name / logo / accent colour are gone.
Browser data was cleared, or you are on a different browser or device. Re-apply via 🎨 Customise — settings must be set once per device/browser.

The logo doesn't appear.
Ensure hotelLogo.jpg is in the same folder as the app. The file name and extension must match exactly (case-sensitive on some systems).

Printing doesn't look right.
In the print dialog, set Margins to None and enable Print backgrounds. For the Sign Generator and Group Check-in, make sure the paper size matches the size selected in the app.

The editor says "No text extracted" when I import a PDF.
The PDF is likely a scanned image rather than a text-based document. A text-based (digitally created) PDF is required. Ask your developer to assist with extraction if you only have a scanned copy.

The app isn't loading my customised items after exporting.
Ensure the exported file is named exactly hotelcompanion_items.js (not hotelcompanion_items (1).js or similar), is in the same folder as the app, and that you reloaded the page after placing it there.

If in doubt, open the browser console (F12 → Console tab) and reload the page. Any loading errors will be shown there with the file name and reason.
